# Layer 6: 実行・矢羽パターンWBS

## このレイヤーの目的

Layer 1〜5 の成果物を、**実行可能な WBS とトレーサビリティマトリクス**に統合する。
矢羽パターン (ハイブリッド型: 全体ウォーターフォール + 並列アジャイルSprint) を標準とし、L2 (矢羽) / L3 (アクティビティ) / L4 (タスク) の 3 階層で展開。

## 推奨実行順序

1. **矢羽構成の決定** - 標準 7 矢羽をベースにプロジェクト特性で調整
2. **L3 アクティビティ展開** - 各矢羽に 3〜8 個の L3 を配置
3. **L4 タスク分解** - 各 L3 に 3〜5 個の L4 を配置
4. **工数見積** - L4 → L3 → L2 の順で積み上げ
5. **トレーサビリティ確立** - ビジネス目標 から L4 タスクまでを貫く

## 矢羽パターン標準構成 (データ分析基盤)

| 矢羽 | 名称 | 進行 | 工数 | 期間 |
|------|------|------|------|------|
| 矢羽① | 業務要件定義 | 直列・ウォーターフォール | 6人月 | 6ヶ月 |
| 矢羽② | PoC + システム要件定義 | 直列・ウォーターフォール | 6人月 | 3ヶ月 |
| 矢羽③ | データ連携開発Sprint | 並列・アジャイル | 7.5人月 | 6ヶ月 |
| 矢羽④ | AI分析・dbt開発Sprint | 並列・アジャイル | 8.5人月 | 6ヶ月 |
| 矢羽⑤ | 検索・BI開発Sprint | 並列・アジャイル | 7人月 | 6ヶ月 |
| 矢羽⑥ | 非機能・運用 | 横断・並走 | 6人月 | 12ヶ月 |
| 矢羽⑦ | 総合テスト・パイロット展開 | 直列・ウォーターフォール | 5人月 | 6ヶ月 |

**進行フロー:** 矢羽① → ② → ③④⑤ (並列) → ⑦ / 矢羽⑥は全期間横断

## 入力 (前レイヤーから受け取るもの)

- Layer 2 ユースケース仮説一覧、ロードマップ
- Layer 3 ADR、C4 コンテナ図
- Layer 4 機能要件 + NFR ランディングゾーン
- Layer 5 リスクレジスタ + Kill基準
- Layer 1 投資判断資料 (予算枠の確認)

## 出力 (最終成果物)

- **矢羽パターンWBS** (L2/L3/L4 Markdown 表)
- **Gantt チャート** (Mermaid 記法)
- **工数見積表** (人月、ロール別)
- **依存マトリクス** (矢羽間 / Sprint間)
- **トレーサビリティマトリクス** (目標↔仮説↔ADR↔要件↔リスク↔タスク)

## 年間1億円規模での粒度指針

| 階層 | 粒度 | 個数目安 | 工数範囲 |
|------|------|---------|---------|
| L2 矢羽 | フェーズレベル | 7 個 | 5〜12 人月 |
| L3 アクティビティ | 成果物単位 | 25〜45 個 | 1〜2 人月 |
| L4 タスク | 1 人で担当可能 | 80〜150 個 | 0.1〜0.5 人月 |

L4 より細かく分解しない (デイリータスクは Sprint チケットの粒度で別管理)。
