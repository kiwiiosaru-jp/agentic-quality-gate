# 個情法対応マトリクス

## 対応する品質評価ID

- **COMPLY-PIPA-JP-001 (Critical)** - 個情法第18-31条の適合性が体系的に確認されていない
- 関連: LEGAL-PII-001、DATA-DELETE-001

## 概要

個人情報保護法 (個情法) における**第18条〜第31条**の主要条文 (利用目的、第三者提供、開示請求等) について、プロジェクトでの遵守状況をマトリクス化する。これは個情委への報告・行政指導対応の基盤となる。

このマトリクスの目的:

1. **網羅性確保** - 個情法の主要条文を全てカバー
2. **適合状況の可視化** - 各条文ごとに 「適合 / 部分適合 / 不適合 / 該当なし」 を明示
3. **是正計画の起点** - 不適合項目を是正タスク化して Layer 6 WBS へ展開
4. **本人請求対応の手順整備** - 開示・訂正・削除請求の対応 SOP の根拠

## いつ使うか

- 矢羽①業務要件定義の中盤 (Week 4-6)
- 個情法改正があった時 (法施行日の 6ヶ月前から準備)
- 個情委から照会・行政指導があった時
- 自己点検監査の年次サイクル

## 個情法の主要条文一覧 (2025年改正対応)

| 条文     | 内容                                             | プロジェクト対応観点                       |
| -------- | ------------------------------------------------ | ------------------------------------------ |
| 第17条   | 利用目的の特定                                   | 利用目的の文言が抽象的すぎないか           |
| 第18条   | 利用目的による制限                               | 当初目的を超える利用がないか               |
| 第19条   | 不適正利用の禁止                                 | 違法・不当行為への利用がないか             |
| 第20条   | 適正取得・要配慮個人情報の取得制限               | 不正手段での取得、要配慮の同意取得         |
| 第21条   | 取得時の利用目的の通知・公表                     | プライバシーポリシーでの公表               |
| 第22条   | データ内容の正確性確保                           | 訂正請求への対応プロセス                   |
| 第23条   | 安全管理措置                                     | 組織的・人的・物理的・技術的措置           |
| 第24条   | 従業者の監督                                     | 従業員教育、退職時の権限剥奪               |
| 第25条   | 委託先の監督                                     | 委託契約書の安全管理条項、監査権           |
| 第26条   | 漏えい等の報告等                                 | 個情委への 3-5 営業日以内報告              |
| 第27条   | 第三者提供の制限                                 | 本人同意 / オプトアウト / 共同利用         |
| 第28条   | 外国にある第三者への提供の制限                   | 越境移転の本人同意 / 基準適合体制          |
| 第29条   | 第三者提供に係る記録の作成等                     | 提供日時、提供先、項目の記録               |
| 第30条   | 第三者提供を受ける際の確認等                     | 受領元の確認、取得経緯の確認               |
| 第31条   | 個人関連情報の第三者提供制限                     | Cookie ID 等の提供時の本人同意確認         |
| 第33条   | 開示請求への対応                                 | 開示手続きの公表、本人確認、料金           |
| 第34条   | 訂正請求への対応                                 | 訂正手続き、訂正期限                       |
| 第35条   | 利用停止・消去請求                               | 削除請求対応 SOP                           |

## 手順

### ステップ 1: 条文ごとの現状把握

各条文について、以下の 4 区分で評価:

| 区分          | 定義                                             |
| ------------- | ------------------------------------------------ |
| 適合          | 対応プロセス・文書・システム実装が完備           |
| 部分適合      | プロセスはあるが文書/実装が一部未整備             |
| 不適合        | 対応プロセスがない、または明確な違反状態         |
| 該当なし      | 当該条文がプロジェクトの取扱範囲外               |

### ステップ 2: マトリクス作成

横軸: 5観点 (取得/利用/保管/提供/本人請求)
縦軸: 個情法条文番号

各セルには「適合状況 + 根拠資料 + 担当者」を記載。

### ステップ 3: 不適合項目の是正計画化

「不適合」「部分適合」と判定された項目について、以下を整理:

- 是正タスク
- 期限
- 担当者
- Layer 5 リスクレジスタへの登録

### ステップ 4: 法務レビュー → 監査対応

- 法務部レビュー (1 週間)
- 内部監査部の追加チェック (任意、推奨)
- 経営層 (CISO/CPO) 承認

## 月次レビューでの確認項目

- 「部分適合」「不適合」項目の進捗
- 是正計画の期限超過の有無
- 個情法改正情報の取込み (個情委ガイドライン更新等)
- 漏えい・本人請求の発生件数とトレンド
- 委託先の安全管理措置の確認 (年次)

## 漏えい時の報告フロー (第26条対応)

```
漏えい検知 (CSIRT または現場)
   ↓ 即時 (最大 24 時間以内)
社内エスカレーション (CISO / CPO / 法務 / 広報)
   ↓ 速やかに (3-5 営業日以内)
個情委への速報 (Web フォーム)
   ↓ 30日以内
個情委への確報 (調査結果、再発防止策)
   ↓ 並行して
本人への通知 (個別通知 or 公表)
```

## pm-blueprint 連携

### Layer 5 リスクレジスタとの紐付け

「不適合」「部分適合」項目は、必ず Layer 5 のリスクレジスタに登録:

```yaml
- id: R-LEG-12
  title: "個情法第28条不適合 (越境移転の本人同意未整備)"
  category: legal
  probability: high
  impact: critical
  description: |
    Snowflake US リージョンへのデータ送信について、
    本人同意取得プロセスが未整備で、個情委から行政指導が入る可能性。
  triggers:
    - 個情委からの照会
    - 内部監査の指摘
  mitigations:
    - リージョン変更 (Japan East)
    - または、本人同意フォームの実装
  owner: "法務部長"
  kill_criterion: |
    個情委から行政指導 (停止命令) が出た場合、
    全機能停止 + 是正計画提出 + 6ヶ月以内の是正完了
```

### 品質ゲートID対応

- **COMPLY-PIPA-JP-001**: 本マトリクスの全条文で「適合」または「該当なし」になれば Critical 解消
- **LEGAL-PII-001**: 第17・18条の整合性は個人情報取扱台帳と相互参照
- **DATA-DELETE-001**: 第35条 (利用停止・消去) はデータ保持_削除戦略.md と相互参照

### Layer 6 WBS への展開

- 1.2.1 個情法主要条文の整理 (法務、3 営業日)
- 1.2.2 各条文の現状ヒアリング (PM + 関係部門、5 営業日)
- 1.2.3 マトリクス v0.1 ドラフト
- 1.2.4 不適合項目の是正計画作成
- 1.2.5 経営層 (CISO/CPO) 承認 (1 週間)
- 1.2.6 是正タスクの WBS 組込み
