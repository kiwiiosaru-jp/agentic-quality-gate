# 業法該当性判定書

## 対応する品質評価ID

- **LEGAL-INDUSTRY-001 (Critical)** - 業種別の業法該当性が判定されていない
- 関連: LEGAL-PII-001、COMPLY-PIPA-JP-001

## 概要

プロジェクトの業務領域に応じて該当しうる**業法 (業種別の業界規制法)** を網羅的に洗い出し、必要な届出・許認可・契約条項を整理する。データ分析基盤・コールセンター・物流業のいずれもプロジェクト主体側 (= 弊社、または顧客企業) が業規制対象になりうる。

このスキルの目的:

1. **適用法令の網羅** - 個情法以外の業法を見落とさない
2. **必要な届出・許認可の特定** - 行政への事前手続きの工数とコストを把握
3. **契約条項の事前整備** - 顧客との契約・委託契約への必要条項組込み
4. **専門家承認** - 弁護士 (業界経験者) の事前承認による訴訟リスク低減

## いつ使うか

- 矢羽①業務要件定義の初期 (Week 1-3)
- 新規業界・新規業種への参入時
- 業法改正があった時 (法施行日の 12ヶ月前から準備)
- M&A で業界特性が変わる時

## 手順

### ステップ 1: 業務領域の特定

プロジェクトの「対象業務」「想定エンドユーザー」「取扱う商品/サービス」を整理。

例:
- 「物流業向けの配送可視化基盤」→ 貨物自動車運送事業法、独禁法 (荷主との関係)
- 「コールセンター向けの VoC 分析」→ 電気通信事業法、特商法、個情法第2条
- 「DWH 内のデータを使った広告配信」→ 個情法第31条、景表法、特商法

### ステップ 2: 該当性判定

各業法について、4 区分で判定:

| 区分             | 定義                                       |
| ---------------- | ------------------------------------------ |
| 直接該当         | 自社が業法の規制対象事業者                 |
| 間接該当         | 委託先・顧客が規制対象、契約条項に影響     |
| 部分該当         | 一部機能/業務範囲のみ規制対象               |
| 該当なし         | 規制対象外                                 |

### ステップ 3: 必要な届出・許認可の整理

「直接該当」と判定された法令について:

- 必要な届出書類
- 提出先行政庁
- 提出までのリードタイム (例: 運送業の許可は 3〜6ヶ月)
- 更新サイクル
- 違反時の罰則

### ステップ 4: 契約条項への反映

顧客契約・委託契約・利用規約に必要な条項を整理:

- 個人情報の取扱条項
- 通信の秘密の遵守条項
- 下請法に基づく発注書の記載事項
- 景表法に基づく広告表示の責任分界

### ステップ 5: 専門家承認フロー

業法分野は弁護士でも専門が分かれるため、以下のフロー:

```
プロジェクト責任者 起票
   ↓
社内法務 一次レビュー
   ↓
業界特化の外部弁護士 (運送業/通信/個情法等)
   ↓
弁護士意見書の発行 (書面)
   ↓
経営層 承認
   ↓
判定書 確定
```

外部弁護士費用は 1業法あたり 30〜50万円が目安 (年間予算 1億円なら 100万円程度を確保)。

## 専門家意見書 (弁護士) のサンプル目次

```
1. 案件概要
2. 関連する業法と該当性判定
3. 必要な届出・許認可
4. 契約条項への反映事項
5. 業務オペレーションへの影響
6. 想定リスクと推奨対策
7. 法令改正の見通し
8. 結論
```

### 品質ゲートID対応

- **LEGAL-INDUSTRY-001**: 全該当業法について判定+対応計画があれば Critical 解消
- **LEGAL-PII-001**: 個情法は本判定書の冒頭で必ず該当性判定する
- **COMPLY-PIPA-JP-001**: 業法と個情法の関係 (例: 通話録音の特殊性) を相互参照

## 注意事項

- 業法は改正頻度が高く、判定書には**判定時点のバージョン**を必ず明記
- 業界特化の弁護士を選定すること (一般民商事の弁護士では業法判定が不正確になりがち)
- 国際展開時は相手国の業法も同時判定が必要 (例: GDPR、PIPA-韓国、PIPL-中国)
- 業法違反は刑事罰を伴うケースもあり、Critical Fail 扱いとして優先対応する
