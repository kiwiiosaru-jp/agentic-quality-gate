# Layer 1: 経営層視点

## このレイヤーの目的

プロジェクト**開始前**および**フェーズゲート**において、経営層の以下 3 判断を支援する:

1. **投資判断** (Go / No-Go): このプロジェクトに投資する価値があるか
2. **ピボット判断** (Pivot): 方向転換するべきか、原案で続行するか
3. **撤退判断** (Exit): 撤退するべきか、継続投資するか

年間 1 億円規模での想定であり、より大規模 (10 億円超) の案件では、別スキル (例: `c-level-advisor`) を併用することを推奨する。

## 本レイヤーの使いどころ

| タイミング | 判断 | 使用テンプレート |
|----------|-----|----------------|
| キックオフ前 | 投資 Go / No-Go | A. 投資判断テンプレート |
| 矢羽①完了時 | 次フェーズ Go / 方向調整 | A / B |
| 矢羽②完了時 | アーキ確定と投資継続 | A / B |
| 月次 Kill基準超過時 | 撤退 or ピボット | B / C |
| 外部環境変化時 | 全体見直し | A / B / C |

## 推奨フロー

```
Layer 2〜5 の出力が揃う
    ↓
Layer 1 で経営判断用の3テンプレートを適用
    ↓
答えるべき問いに答える
    ↓
経営層へ提出 → 判断取得
```

## 入力 (前レイヤーから受け取るもの)

- Layer 2 のユースケース仮説、前提 (特に Viability 軸)
- Layer 3 の ADR (主要投資の内訳)
- Layer 4 の NFR (SLA 定義)
- Layer 5 のリスクレジスタ + Kill基準
- Layer 6 の WBS と工数・コスト試算

## 出力 (経営層への資料)

- **投資判断資料** (ROI, シナリオ別期待値, Go/No-Go 根拠)
- **ピボット / 撤退判断資料** (Kill基準照合, 代替案評価)
- **リスクエスカレーションパッケージ**

## 年間1億円規模での軽量化方針

- 投資判断テンプレートは **A4 3〜5 枚**で経営層に提示できる粒度
- NPV 等の厳密な財務計算は 1 回目は不要 (定性 + 大まかな定量で十分)
- ピボット判断は Kill基準との**客観的照合**を中心に
- 撤退判断は **Sunk cost 無視**の原則を徹底

## 注意事項

- 経営層は「データを見て判断する」より「ストーリーで判断する」傾向が強い。資料は定量と定性の両方を揃える
- 投資承認後も、矢羽①完了時に再度 Go/No-Go を問う「スモール Go」を推奨
- 経営層の意思決定には根回しが効く。`custom/日本型PMコンテキスト.md` も参照
